## 1. 项目概述

### 1.1 项目背景

WikiHow 知识库系统是一个仿照 WikiHow 网站设计的知识分享平台，旨在为用户提供一个创作和浏览步骤式图文教程的在线社区。系统采用前后端分离架构，支持用户注册登录、文章创作、互动交流等完整功能。

### 1.2 项目目标

- ✅ 实现步骤式图文教程的创作与展示（WikiHow 核心特色）
- ✅ 提供完整的用户认证与权限管理体系
- ✅ 支持文章版本管理与内容审核机制
- ✅ 实现编辑冲突调解与过时内容标记
- ✅ 提供丰富的互动功能（评论、点赞、收藏、通知）
- ✅ 支持离线查看与 PWA 功能

## 3. 核心功能详解

### 3.1 用户认证与权限管理

#### 3.1.1 功能概述

实现了完整的用户认证体系，支持四种用户角色：游客、普通用户、审核员、管理员。

#### 3.1.2 技术实现

**JWT 认证流程**:

```
1. 用户登录 → 后端验证用户名密码
2. 验证成功 → 生成 JWT Token (包含 userId, username, role)
3. 返回 Token → 前端存储到 localStorage
4. 后续请求 → 请求头携带 Authorization: Bearer {token}
5. 后端拦截 → JwtAuthenticationFilter 验证 Token
6. 解析成功 → 将用户信息存入 ThreadLocal (UserContext)
7. 业务处理 → 通过 UserContext.getCurrentUserId() 获取当前用户
```

**核心代码**:

```java
// JWT 工具类
public class JwtUtil {
    public static String generateToken(Long userId, String username, String role) {
        return Jwts.builder()
            .setSubject(userId.toString())
            .claim("username", username)
            .claim("role", role)
            .setIssuedAt(new Date())
            .setExpiration(new Date(System.currentTimeMillis() + EXPIRATION))
            .signWith(SignatureAlgorithm.HS512, SECRET_KEY)
            .compact();
    }
}

// JWT 拦截器
@Component
public class JwtAuthenticationFilter implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request, 
                            HttpServletResponse response, 
                            Object handler) {
        String token = request.getHeader("Authorization");
        if (token != null && token.startsWith("Bearer ")) {
            String jwt = token.substring(7);
            Claims claims = JwtUtil.parseToken(jwt);
            Long userId = Long.parseLong(claims.getSubject());
            UserContext.setCurrentUserId(userId);
            return true;
        }
        throw new BusinessException(ResultCode.UNAUTHORIZED);
    }
}
```

#### 3.1.3 权限控制

- **游客**: 仅可浏览文章列表和详情
- **普通用户**: 可创建文章、评论、点赞、收藏
- **审核员**: 可审核文章、处理过时举报
- **管理员**: 拥有所有权限，可管理用户、文章、分类

### 3.2 文章管理系统

#### 3.2.1 功能概述

文章是系统的核心实体，支持步骤式图文教程的创作、编辑、展示。

#### 3.2.2 文章数据结构

```java
Article (文章主体)
├── id: 文章ID
├── title: 标题
├── summary: 摘要
├── coverImage: 封面图
├── userId: 作者ID
├── categoryId: 分类ID
├── tags: 标签 (逗号分隔)
├── difficulty: 难度 (EASY/MEDIUM/HARD)
├── status: 状态 (0草稿/1待审核/2已发布/3已拒绝/4已下架)
├── viewCount: 浏览量
├── likeCount: 点赞数
├── favoriteCount: 收藏数
├── commentCount: 评论数
├── isOutdated: 是否过时
├── outdatedReason: 过时原因
├── references: 参考文献 (JSON)
└── steps: List<ArticleStep> (步骤列表)
    ├── stepOrder: 步骤序号
    ├── title: 步骤标题
    ├── content: 步骤内容
    └── image: 步骤图片
```

#### 3.2.3 文章创建流程

```
1. 用户填写文章基本信息 (标题、摘要、分类、难度、标签)
2. 上传封面图 → 调用文件上传接口 → 返回图片 URL
3. 添加步骤 → 使用 StepEditor 组件
   ├── 填写步骤标题和内容
   ├── 上传步骤图片
   └── 支持拖拽排序
4. 可选添加参考文献
5. 点击发布 → 提交到后端
6. 后端保存文章 → 保存步骤 → 返回文章ID
7. 文章状态设为"待审核" (status=1)
8. 审核员审核通过后 → 状态变为"已发布" (status=2)
```

#### 3.2.4 文章展示流程

```
1. 用户访问文章详情页 → GET /api/articles/{id}
2. 后端查询文章信息 + 步骤列表
3. 增加浏览量 (viewCount++)
4. 如果用户已登录 → 查询是否已点赞/收藏
5. 返回完整文章数据 (包含作者信息、分类信息、步骤列表)
6. 前端渲染:
   ├── 文章头部 (标题、作者、分类、难度、统计数据)
   ├── 步骤列表 (WikiHow 风格的步骤式布局)
   ├── 互动按钮 (点赞、收藏、分享)
   ├── 参考文献区域
   ├── "你也许感兴趣" 推荐区域
   └── 评论区
```

### 3.3 版本管理系统

#### 3.3.1 功能概述

实现了完整的文章版本管理功能，记录每次编辑的历史版本，支持版本对比和回滚。

#### 3.3.2 版本管理流程

```
1. 用户编辑文章 → 提交更新
2. 后端在更新文章前 → 自动创建版本快照
3. 版本记录包含:
   ├── version_number: 版本号 (自动递增)
   ├── title: 标题快照
   ├── summary: 摘要快照
   ├── content_snapshot: 步骤内容快照 (JSON 格式)
   ├── user_id: 编辑者ID
   ├── change_description: 修改说明
   └── created_at: 创建时间
4. 用户可查看版本历史 → GET /api/articles/{id}/versions
5. 用户可查看指定版本 → GET /api/articles/{id}/versions/{versionId}
6. 用户可回滚到指定版本 → POST /api/articles/{id}/revert/{versionId}
```

#### 3.3.3 核心代码

```java
@Service
public class ArticleVersionServiceImpl implements ArticleVersionService {
    
    @Transactional
    public void saveVersion(Long articleId, String changeDescription) {
        // 查询当前文章
        Article article = articleMapper.selectById(articleId);
        // 查询步骤列表
        List<ArticleStep> steps = articleStepMapper.selectByArticleId(articleId);
        
        // 获取最新版本号
        Integer latestVersion = articleVersionMapper.getLatestVersion(articleId);
        int newVersion = (latestVersion == null) ? 1 : latestVersion + 1;
        
        // 创建版本记录
        ArticleVersion version = new ArticleVersion();
        version.setArticleId(articleId);
        version.setVersionNumber(newVersion);
        version.setTitle(article.getTitle());
        version.setSummary(article.getSummary());
        version.setContentSnapshot(JSON.toJSONString(steps)); // 步骤快照
        version.setUserId(UserContext.getCurrentUserId());
        version.setChangeDescription(changeDescription);
        
        articleVersionMapper.insert(version);
    }
}
```

### 3.5 编辑冲突调解

#### 3.5.1 功能概述

使用编辑锁机制，防止多人同时编辑同一篇文章导致的冲突。

#### 3.5.2 编辑锁流程

```
1. 用户A进入编辑页面
   ↓
2. 前端请求获取编辑锁 → POST /api/articles/{id}/lock
   ↓
3. 后端检查是否已被锁定
   ├─ 未锁定 → 创建锁记录，返回成功
   └─ 已锁定 → 返回锁定信息 (谁在编辑、预计解锁时间)
   ↓
4. 用户A获得锁 → 可以编辑
   ↓
5. 用户B进入编辑页面 → 请求获取锁
   ↓
6. 后端返回: "该文章正在被 xxx 编辑，预计 xx:xx 解锁"
   ↓
7. 用户B看到提示 → 无法编辑，只能查看
   ↓
8. 用户A编辑完成 → 离开页面 → 自动释放锁
   或 30分钟后自动过期
```

#### 3.5.3 锁续期机制

```javascript

  if (isEditing) {
    renewEditLock(articleId)
  }
}, 5 * 60 * 1000)
```

```java
@RestController
@RequestMapping("/api/files")
@RequiredArgsConstructor
public class FileController {
    
    @Value("${file.upload.path}")
    private String uploadPath;
    
    @PostMapping("/upload")
    public Result<Map<String, String>> upload(@RequestParam("file") MultipartFile file) {
        // 验证文件类型
        String contentType = file.getContentType();
        if (!contentType.startsWith("image/")) {
            throw new BusinessException("只能上传图片文件");
        }
        
        // 验证文件大小 (最大 5MB)

        // 生成唯一文件名
        String originalFilename = file.getOriginalFilename();
        String extension = originalFilename.substring(originalFilename.lastIndexOf("."));
        String filename = UUID.randomUUID().toString() + extension;
        
        // 保存文件
        Path path = Paths.get(uploadPath, filename);
        Files.copy(file.getInputStream(), path);
        
        // 返回访问 URL
        String url = "/uploads/" + filename;
        return Result.success(Map.of("url", url));
    }
```

```
  ├─1:N─→ article_version (版本)
  ├─1:N─→ article_review (审核)
  ├─1:1─→ article_edit_lock (编辑锁)
  ├─1:N─→ comment (评论)
  ├─1:N─→ like_record (点赞)
  ├─1:N─→ favorite (收藏)
  └─1:N─→ outdated_report (过时举报)

comment (评论)
  └─1:N─→ comment (回复)
```

### 6.3 索引设计

为提高查询性能，设计了以下索引:

```sql
-- 用户表
CREATE UNIQUE INDEX uk_username ON user(username);
CREATE UNIQUE INDEX uk_email ON user(email);

-- 文章表
CREATE INDEX idx_user ON article(user_id);
CREATE INDEX idx_category ON article(category_id);
CREATE INDEX idx_created ON article(created_at);
CREATE INDEX idx_status ON article(status);

-- 文章步骤表
CREATE INDEX idx_article ON article_step(article_id);

-- 评论表
CREATE INDEX idx_article ON comment(article_id);
CREATE INDEX idx_user ON comment(user_id);
CREATE INDEX idx_parent ON comment(parent_id);

-- 点赞表
CREATE UNIQUE INDEX uk_user_article ON like_record(user_id, article_id);
CREATE INDEX idx_article ON like_record(article_id);

-- 收藏表
CREATE UNIQUE INDEX uk_user_article ON favorite(user_id, article_id);
CREATE INDEX idx_article ON favorite(article_id);
```

### 10.2 技术亮点

#### 10.2.1 后端技术亮点

1. **统一响应格式**: 所有 API 返回统一的 `Result<T>` 格式，便于前端统一处理
2. **全局异常处理**: 使用 `@ControllerAdvice` 统一捕获和处理异常
3. **参数校验**: 使用 Jakarta Validation 进行参数校验，减少重复代码
4. **MyBatis-Plus 增强**: 简化 CRUD 操作，提高开发效率
5. **JWT 认证**: 无状态认证，支持分布式部署
6. **ThreadLocal 用户上下文**: 优雅地获取当前登录用户信息
7. **事务管理**: 使用 `@Transactional` 确保数据一致性
8. **DTO 转换模式**: 使用静态方法进行 Entity 到 DTO 的转换

#### 10.2.2 前端技术亮点

1. **Composition API**: 使用 Vue 3 Composition API，代码更简洁
2. **Pinia 状态管理**: 轻量级状态管理，支持持久化
3. **Axios 拦截器**: 统一处理请求和响应，自动添加 Token
4. **路由守卫**: 实现路由级别的权限控制
5. **组件化开发**: 高度组件化，提高代码复用性
6. **PWA 支持**: 支持离线查看，提升用户体验
7. **响应式设计**: 适配不同屏幕尺寸

#### 10.2.3 数据库设计亮点

1. **合理的表结构**: 15 张表，职责清晰，避免冗余
2. **索引优化**: 为常用查询字段添加索引，提高查询性能
3. **外键约束**: 使用外键保证数据完整性
4. **JSON 字段**: 使用 JSON 存储复杂数据 (步骤快照、参考文献)
5. **软删除**: 重要数据使用状态字段而非物理删除

### 10.3 项目优势

| Vite         | 新一代构建工具     | 启动快，热更新快           |
| Axios        | 主流 HTTP 客户端   | 拦截器强大，易于封装       |

### 10.6 开发经验总结

#### 10.6.1 成功经验

1. **需求先行**: 先完成需求文档、设计文档，再开始编码
2. **分层开发**: 严格按照分层架构开发，职责清晰
3. **接口先行**: 先定义接口，前后端并行开发
4. **增量开发**: 按模块逐步开发，每个模块完成后测试
5. **代码复用**: 抽取公共组件和工具类，提高复用性

#### 10.6.2 遇到的问题与解决

**问题 1: 编辑冲突**
- 问题: 多人同时编辑同一篇文章导致数据覆盖
- 解决: 实现编辑锁机制，同一时间只允许一人编辑

**问题 2: 版本管理**
- 问题: 文章修改后无法追溯历史版本
- 解决: 实现版本快照功能，每次修改自动保存版本

**问题 3: 通知推送**
- 问题: 用户无法及时收到评论、审核等通知
- 解决: 实现通知系统，关键操作自动发送通知

**问题 4: 过时内容**
- 问题: 文章内容过时但无法标记
- 解决: 实现过时举报和标记功能

**问题 5: 权限控制**
- 问题: 不同角色的权限管理复杂
- 解决: 实现基于角色的权限控制，使用拦截器统一处理
